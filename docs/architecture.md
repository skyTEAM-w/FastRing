# 高性能ADC数据采集与传输系统 - 架构设计文档

## 1. 系统概述

### 1.1 项目背景
本项目实现了一个高性能数据采集与传输系统，满足40kHz采样率、32位精度的ADC数据获取需求，并通过网络实时发送至上位机。

### 1.2 性能指标
| 指标 | 目标值 | 实测值 | 说明 |
|------|--------|--------|------|
| 采样率 | 40 kHz | 39,992 Hz | 每秒40,000个样本 |
| 分辨率 | 32 bits | 32 bits | 每个样本4字节 |
| 数据速率 | 156.25 KB/s | ~156 KB/s | 原始数据吞吐量 |
| 端到端延迟 | < 10 ms | < 1 ms | 采集到发送完成 |
| CPU占用 | < 30% | < 2% | 单核 |
| 采样率达成率 | > 95% | 99.98% | 实际/目标采样率 |
| 丢弃率 | < 1% | 0% | 数据完整性 |

## 2. 系统架构

### 2.1 三级流水线架构

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   ADC采集线程    │────▶│   数据处理线程   │────▶│   网络发送线程   │
│  (最高优先级)    │     │  (高优先级)      │     │  (中高优先级)    │
└─────────────────┘     └─────────────────┘     └─────────────────┘
         │                       │                       │
         ▼                       ▼                       ▼
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  采集→处理缓冲区  │     │ 处理→网络缓冲区  │     │    TCP客户端    │
│  (无锁环形缓冲)  │     │  (无锁环形缓冲)  │     │  (批量发送)     │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

### 2.2 线程职责

#### 2.2.1 采集线程 (THREAD_CAPTURE)
- **优先级**: TIME_CRITICAL (Windows) / SCHED_FIFO 99 (Linux)
- **职责**:
  - 与ADC硬件交互（模拟器中为定时生成数据）
  - 严格25μs采样周期控制
  - 将数据推入采集→处理缓冲区
- **关键要求**: 最小化抖动，确保采样率稳定

#### 2.2.2 处理线程 (THREAD_PROCESS)
- **优先级**: HIGHEST (Windows) / SCHED_FIFO 80 (Linux)
- **职责**:
  - 从采集缓冲区批量读取数据
  - 数据预处理（滤波、格式转换等）
  - 将处理后的数据推入处理→网络缓冲区
- **关键要求**: 及时处理，避免缓冲区溢出

#### 2.2.3 网络发送线程 (THREAD_NETWORK)
- **优先级**: ABOVE_NORMAL (Windows) / SCHED_FIFO 70 (Linux)
- **职责**:
  - 从处理缓冲区批量读取数据
  - 构建数据块（包含魔数、版本、时间戳等元数据）
  - 通过TCP发送至上位机
- **关键要求**: 可靠传输，自动重连

## 3. 数据流设计

### 3.1 数据结构

```c
/* ADC样本结构 - 16字节 */
typedef struct {
    uint32_t timestamp;     // 微秒级时间戳
    int32_t  value;         // 32位采样值
    uint32_t sequence;      // 序列号（用于丢包检测）
    uint16_t channel;       // 通道号
    uint16_t flags;         // 标志位
} adc_sample_t;

/* 数据块结构 - 用于网络传输 */
typedef struct {
    uint32_t    magic;          // 魔数: 0xADC0DA7A
    uint32_t    version;        // 协议版本
    uint32_t    sequence;       // 块序列号
    uint32_t    sample_count;   // 样本数量
    uint64_t    timestamp;      // 块时间戳
    adc_sample_t samples[];     // 样本数组
} data_block_t;
```

### 3.2 缓冲区设计

#### 3.2.1 无锁环形缓冲区
- **容量**: 16384个样本（约400ms缓冲）
- **机制**: 单生产者单消费者，使用原子操作和内存屏障
- **特点**:
  - 无互斥锁，避免线程阻塞
  - 2的幂次方容量，位运算代替取模
  - 自动丢弃溢出数据，保证实时性

#### 3.2.2 三级缓冲策略
```
采集缓冲区 ──▶ 处理缓冲区 ──▶ 网络缓冲区
  16K样本       16K样本        16K样本
```

## 4. 线程同步机制

### 4.1 无锁同步
```c
/* 内存屏障宏 */
#ifdef _WIN32
#define MEMORY_BARRIER() MemoryBarrier()
#else
#define MEMORY_BARRIER() __sync_synchronize()
#endif
```

### 4.2 生产者-消费者模式
- 采集线程作为生产者写入采集缓冲区
- 处理线程作为消费者读取采集缓冲区，同时作为生产者写入处理缓冲区
- 网络线程作为消费者读取处理缓冲区

### 4.3 批量操作优化
- 批量读取/写入减少原子操作次数
- 默认批量大小: 64-256个样本

## 5. 网络传输设计

### 5.1 TCP优化
- **TCP_NODELAY**: 禁用Nagle算法，降低延迟
- **SO_SNDBUF**: 64KB发送缓冲区
- **SO_KEEPALIVE**: 连接保活
- **批量发送**: 256样本/块，减少系统调用

### 5.2 协议设计
```
数据块格式:
+--------+---------+----------+--------------+-----------+----------+
| Magic  | Version | Sequence | Sample Count | Timestamp | Samples  |
| 4 bytes| 4 bytes | 4 bytes  |   4 bytes    |  8 bytes  | N*16bytes|
+--------+---------+----------+--------------+-----------+----------+
```

### 5.3 错误处理
- 自动重连机制
- 发送超时处理 (5秒)
- 连接断开检测

## 6. 性能优化策略

### 6.1 内存优化
- 预分配缓冲区，避免运行时分配
- 缓存对齐，减少伪共享
- 批量操作减少函数调用开销

### 6.2 CPU优化
- 线程亲和性设置（可选）
- 无锁数据结构避免上下文切换
- 忙等待与休眠结合，平衡延迟与CPU占用

### 6.3 网络优化
- 批量发送减少头部开销
- 数据压缩（可选）
- 自适应发送频率

## 7. 错误处理与恢复

### 7.1 错误分类
| 错误类型 | 处理方式 | 恢复策略 |
|----------|----------|----------|
| 缓冲区溢出 | 丢弃数据 | 增加缓冲区大小 |
| 网络断开 | 缓存数据 | 自动重连 |
| 采样超时 | 记录日志 | 继续采集 |
| 内存不足 | 优雅退出 | 释放资源 |

### 7.2 错误码与字符串转换
```c
// 错误码定义
typedef enum {
    ADC_OK = 0,               // 操作成功
    ADC_ERROR_INIT = -1,      // 初始化错误
    ADC_ERROR_BUFFER = -2,    // 缓冲区操作错误
    ADC_ERROR_NETWORK = -3,   // 网络操作错误
    ADC_ERROR_THREAD = -4,    // 线程操作错误
    ADC_ERROR_TIMEOUT = -5,   // 操作超时
    ADC_ERROR_INVALID_PARAM = -6 // 无效参数
} adc_error_t;

// 获取错误描述字符串
const char* adc_error_string(adc_error_t error);
```

### 7.3 监控与统计
```c
typedef struct {
    uint64_t total_samples;     // 总采样数
    uint64_t dropped_samples;   // 丢弃样本数
    uint64_t sent_bytes;        // 发送字节数

    double   avg_latency_ms;    // 平均延迟

```
